# Linter MCP Server

An MCP server that bridges **VS Code Diagnostics** (Errors, Warnings, Information, Hints) to the AID agent system. Unlike other servers that run CLI tools, this server communicates with the running VS Code instance via a local WebSocket bridge.

## Architecture

This server connects to a WebSocket server embedded in the **AID Extension** at `ws://localhost:8999`. It retrieves live diagnostics generated by VS Code's language servers (e.g., C/C++ extension).

## Prerequisites

- **Python 3.12+** (managed via `uv`)
- **AID VS Code Extension** running (host bridge)
- Port **8999** available for the WebSocket bridge

## Tools

### Diagnostics

- `get_diagnostics(file_path)`: Get all diagnostics for a specific file.
- `get_diagnostics_by_severity(severity, file_path?)`: Filter by `Error`, `Warning`, `Information`, or `Hint`.
- `get_workspace_diagnostics_summary()`: Overview of total errors/warnings across the workspace.
- `get_active_file_diagnostics()`: Get diagnostics for the currently focused editor.

### Connection

- `check_bridge_connection()`: Verify connectivity to the VS Code extension.

## Usage

Start the server:

```bash
uv run --with fastmcp --with websockets fastmcp run linter/main.py -t http -p 8005 -- --vscode-ws-port 8999
```

## Troubleshooting

- **"Bridge not connected"**: Ensure the AID extension is active in VS Code.
- **No Diagnostics**: Diagnostics are only available if VS Code has computed them (e.g., file must be opened or built).
